# 소유권 규칙

1. 특별한 언급이 없는 이상 함수는 인자로부터 소유권을 넘겨받지 않습니다.
2. 1과 2를 위반할 경우에는 Out(), Move()메크로를 이용합니다.
3. Borrow 메크로가 있는 경우에는 함수는 인자의 내용을 수정할 수 없습니다.
4. C언어의 단일 반환값 대신 포인터 변수를 받아 수정하는 변수는 Out메크로 사용합니다.

6. 자기 자신 메모리 해제

# 예외 처리

1. try로 시작하는 함수는 실패 시 약속된 반환값을 반환합니다. 구체적인 코드와 메시지는 인자로 전달 받은 에러 컨텍스트에 추가합니다.
2. try로 시작하지 않고 일반 값을 반환하는 함수는 실패 시 반환값이 정의되지 않습니다.(일반적으로 zero-value).이 때문에 반드시 errContext의 hasError를 호출해야 합니다. 
3. create_계열 함수는 실패시 NULL을 반환하고 구체적인 코드와 메시지는 인자로 전달받은 에러 컨텍스트에 추가합니다.

에러를 반환받은 함수는 다음 세 동작 중 반드시 하나를 수행해야 합니다.

1. 전파: append함수를 호출해 에러 컨텍스트에 자신의 호출 정보를 포함해 상위 계층으로 전달
2. 처리: 해당 에러를 처리하고 복구 시도. 이때 반드시 consume함수를 호출해서 자신이 에러를 처리했다는 사실을 알려줘야 합니다.
3. 단, EOF같은 state는 예외로 합니다.
   
# 이름 규칙

1. 특정 구조체에 속하는 함수(메서드)는 {기능}_{구조체이름}으로 합니다. 언더바 하나로 구분합니다.
2. 구조체 이름은 파스칼케이스, 함수, 지역변수, 필드는 카멜 케이스를 사용합니다.
